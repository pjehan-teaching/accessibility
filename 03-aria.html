ARIA

Accessible Rich Internet Applications

Définition

ARIA, ou Accessible Rich Internet Applications, est une spécification du W3C qui vise à améliorer l'accessibilité des applications web dynamiques. Elle permet aux développeurs d'ajouter des attributs spécifiques aux éléments HTML pour fournir des informations supplémentaires

Ces attributs peuvent être catégorisés en deux types principaux :

  • Rôles : Ils définissent le type d'élément et son comportement, comme un bouton, une boîte de dialogue, etc.
  • Attributs : Ils sont utilisés pour décrire les états et les propriétés des éléments interactifs comme les champs de formulaire, les menus, etc.

Rôles

Les rôles ARIA permettent d'ajouter des informations sémantiques aux éléments HTML, ce qui aide les technologies d'assistance à comprendre la structure et le comportement des éléments interactifs.

Certaines balises HTML ont des rôles ARIA prédéfinis, comme <button>, <nav>, ou <form>. Dans ces cas, les rôles ARIA sont implicites et n'ont pas besoin d'être spécifiés explicitement.

Cependant, pour les éléments HTML qui n'ont pas de rôle ARIA prédéfini, il est important de spécifier le rôle ARIA approprié pour garantir une accessibilité optimale.

Les rôles ARIA peuvent être catégorisés en six catégories.

MDN - ARIA Roles Reference

Rôles - Structure d'un document

Les rôles de structure d'un document aident à organiser le contenu d'une section d'une page web. Ils sont essentiels pour la navigation et la compréhension de la hiérarchie du contenu.

La majorité des éléments HTML permettent aujourd'hui de définir des rôles ARIA implicites et il n'est donc pas nécessaire de les spécifier explicitement. Par exemple, les éléments <article>, <table>, <img>... permettent de structurer le document sans avoir besoin d'ajouter des rôles ARIA.

MDN - Document structure roles

Rôles - Structure d'un document

Cependant, certains éléments HTML n'existent pas ou ne sont pas suffisamment sémantiques pour représenter des éléments spécifiques d'un document.

Par exemple, il n'existe pas d'élément HTML pour représenter un composant de type "tooltip". Dans ce cas, il est nécessaire d'utiliser un rôle ARIA explicite pour indiquer la fonction de l'élément : 


<button aria-describedby="buttontooltip">Bouton</button>
<div role="tooltip" id="buttontooltip">
    
Contenu du tooltip

</div>

Bien entendu, il faudra ajouter du code CSS pour gérer l'affichage du tooltip.

De plus, le bouton devra être accessible au clavier (tabulation) et le tooltip devra pouvoir être fermé par l'utilisateur en appuyant sur la touche Échap ce qui impliquera d'écrire du code JavaScript complémentaire.

Rôles - Structure d'un document

Voici quelques exemples de rôles ARIA pour la structure d'un document :

  • role="toolbar" : Représente une barre d'outils contenant des contrôles. Par exemple, une barre d'outils de mise en forme dans un éditeur de texte.
  • role="feed" : Représente un flux d'actualités ou de contenu.
  • role="math" : Représente une expression mathématique.
  • role="presentation" / role="none" : Indique que l'élément est utilisé uniquement pour la présentation et n'a pas de signification sémantique.
  • role="note" : Représente une note ou un commentaire dans le document.

Rôles - Widget

Les rôles de type "widget" sont utilisés pour décrire des éléments interactifs. Ils sont relativement similaires aux rôles de structure du document, mais seront interactifs et nécessiteront généralement l'utilisation de JavaScript.

De nouveaux, il est préférable d'utiliser les éléments HTML qui ont des rôles ARIA implicites, comme <button>, <progressbar>, <checkbox>, etc.

MDN - Widget roles

Rôles - Widget

Cependant, il existe des cas où les éléments HTML ne sont pas suffisants pour représenter des widgets spécifiques. Dans ces cas, il est nécessaire d'utiliser des rôles ARIA explicites.

Par exemple, si vous créez un composant de type "slider" (curseur), vous pouvez utiliser les rôles ARIA role="tab", role="tablist" et role="tabpanel" pour indiquer qu'il s'agit d'une navigation par onglets. 

<div role="tablist">
    <button role="tab" aria-selected="true" aria-controls="panel-1">Onglet 1</button>
    <button role="tab" aria-controls="panel-2">Onglet 2</button>
</div>
<div id="panel-1" role="tabpanel" aria-labelledby="tab-1">
    Contenu de l'onglet 1
</div>
<div id="panel-2" role="tabpanel" aria-labelledby="tab-2">
    Contenu de l'onglet 2
</div>
MDN - Tab role example

Rôles - Widget

Voici quelques exemples de rôles ARIA pour les widgets :

  • role="menu" : Représente un menu déroulant (préférer la balise <nav> pour les menus de navigation).
  • role="tree" : Représente une structure arborescente, comme un explorateur de fichiers.
  • role="separator" : Permet de séparer visuellement des éléments d'une page (préférer la balise <hr> pour les séparateurs horizontaux). Peut être utilisé par exemple sur une image.
  • role="switch" : Représente un interrupteur à bascule (préférer la balise <input type="checkbox"> pour les cases à cocher).
  • role="slider" : Représente un curseur pour sélectionner une valeur dans une plage (préférer la balise <input type="range">).

Rôles - Repères

Les rôles de type "repères" sont utilisés pour définir la structure d'une page web.

La plupart des éléments HTML existants permettant de définir des rôles ARIA implicites, comme <header>, <footer>, <main>, <section>... Il n'est donc pas nécessaire de les spécifier explicitement.

MDN - Landmark roles

Rôles - Régions dynamiques

Les rôles de type "régions dynamiques" (live regions) sont utilisés pour indiquer des sections de contenu qui peuvent changer dynamiquement, comme les notifications, les alertes ou les messages d'erreur.

Les personnes malvoyantes ou aveugles peuvent être ainsi informées des changements de contenu.

Il n'existe pas d'élément HTML spécifique pour les régions dynamiques, et il sera donc nécessaire d'utiliser des rôles ARIA explicites pour indiquer la nature dynamique du contenu.

MDN - Live regions MDN - Live regions guide

Rôles - Régions dynamiques

Voici quelques exemples de rôles ARIA pour les régions dynamiques :

  • role="alert" : Représente une alerte ou un message d'erreur important.
  • role="log" : Représente un journal d'événements ou de messages.
  • role="marquee" : Similaire au rôle "log", mais les informations ne sont pas ordonnées par date.
  • role="status" : Représente un message d'état ou de progression.
  • role="timer" : Représente un compte à rebours ou un minuteur.

Rôles - Fenêtre

Les rôles de type "fenêtre" (window roles) sont utilisés pour décrire des éléments qui se comportent comme des dialogues dans une application web.

Il n'existe que deux rôles ARIA de type "fenêtre" :

  • role="dialog" : Représente une boîte de dialogue modale ou non modale.
  • role="alertdialog" : Représente une boîte de dialogue modale qui nécessite une attention immédiate et nécessite une action de l'utilisateur.

Il est possible d'utiliser la balise HTML <dialog> pour créer des dialogues.

Attributs

Les attributs ARIA sont utilisés pour fournir des informations supplémentaires sur les éléments interactifs et leur état. Ils sont essentiels pour aider les technologies d'assistance à comprendre le comportement des éléments et à fournir une expérience utilisateur accessible.

Les rôles ARIA peuvent être catégorisés en quatre catégories :

  • Widget : Attributs qui décrivent les éléments interactifs, comme les boutons, les cases à cocher, etc.
  • Région dynamique : Attributs qui décrivent les sections de contenu qui peuvent changer dynamiquement, comme les notifications, les alertes, etc.
  • Drag and drop : Attributs qui décrivent les éléments pouvant être glissés et déposés, comme les listes réordonnables, les tableaux, etc.
  • Relation : Attributs qui décrivent les relations entre les éléments, comme les labels associés aux champs de formulaire, les lignes d'un tableau, etc.

Attributs - Widget

Les attributs ARIA de type "widget" sont utilisés pour décrire les éléments interactifs et leur état. Ils sont essentiels pour aider les technologies d'assistance à comprendre le comportement des éléments et à fournir une expérience utilisateur accessible.

Voici quelques exemples d'attributs ARIA de type "widget" :

  • aria-expanded : Indique si un élément est développé ou réduit.
  • aria-hidden : Indique si un élément est visible ou caché.
  • aria-label : Fournit une étiquette textuelle pour un élément. 
    <button aria-label="Close">x</button>
MDN - Widget attributes

Attributs - Régions dynamiques

Les attributs ARIA de type "régions dynamiques" sont utilisés pour décrire les sections de contenu qui peuvent changer dynamiquement, comme les notifications, les alertes ou les messages d'erreur.

Il existe quatre attributs ARIA pour les régions dynamiques :

  • aria-busy : Indique si un élément est en cours de traitement ou non.
  • aria-live : Indique le niveau de priorité des mises à jour du contenu. Par exemple, aria-live="polite" indique que les mises à jour doivent être annoncées lorsque l'utilisateur n'est pas occupé. Les valeurs possibles sont off, polite et assertive.
  • aria-relevant : Indique quels types de changements (ajout/suppression) dans le contenu doivent être annoncés. Les valeurs possibles sont additions, removals, text et all.
  • aria-atomic : Indique si les mises à jour du contenu doivent être annoncées en tant qu'unité atomique ou non.
MDN - Live region attributes

Attributs - Régions dynamiques

Voici un exemple d'utilisation des attributs ARIA de type "régions dynamiques" : 

<p aria-live="polite">
    Prix total : <span id="total-price">0</span> €
</p>

Dans cet exemple, le paragraphe contenant le prix total est marqué comme une région dynamique avec aria-live="polite". Cela signifie que les mises à jour du prix total seront annoncées lorsque l'utilisateur n'est pas occupé.

Cependant, seul le texte mis à jour sera annoncé ce qui peut être déroutant pour l'utilisateur.

Par exemple, si le prix total passe de 0 € à 10 €, l'utilisateur entendra : 10

Attributs - Régions dynamiques

Pour éviter cela, il est préférable d'utiliser l'attribut aria-atomic="true" pour indiquer que le contenu doit être annoncé en tant qu'unité atomique : 

<p aria-live="polite" aria-atomic="true">
    Prix total : <span id="total-price">0</span> €
</p>

Dans cet exemple, l'ensemble du paragraphe sera annoncé lorsque le prix total change, ce qui est plus cohérent pour l'utilisateur.

À présent, si le prix total passe de 0 € à 10 €, l'utilisateur entendra : Prix total : 10 €

Attributs - Drag and drop

Les attributs ARIA de type "drag and drop" sont utilisés pour décrire les éléments pouvant être glissés et déposés, comme les listes réordonnables, les tableaux, etc.

Il existe deux attributs ARIA pour les éléments de type "drag and drop" :

  • aria-grabbed : Indique si un élément est actuellement saisi pour être déplacé.
  • aria-dropeffect : Indique l'effet de dépôt attendu lors du glissement d'un élément sur une zone de dépôt.

Cependant, ces deux attributs sont maintenant obsolètes et ne doivent plus être utilisés.

MDN - Drag and drop attributes

Attributs - Relation

Les attributs ARIA de type "relation" sont utilisés pour décrire les relations entre les éléments, comme les labels associés aux champs de formulaire, les lignes d'un tableau, etc.

Voici quelques exemples d'attributs ARIA de type "relation" :

  • aria-controls : Indique l'ID d'un élément contrôlé par l'élément actuel. Par exemple, un bouton permettant d'afficher un onglet de contenu.
  • aria-labelledby : Indique l'ID d'un élément qui sert de label pour l'élément actuel. 
    <h2 id="heading">Comment utiliser les ARIA attributes<h2>
<button aria-labelledby="heading">Lire la suite</button>
  • aria-describedby : Indique l'ID d'un élément qui décrit l'élément actuel. Cet attribut est similaire à aria-labelledby, mais il est utilisé pour fournir des informations supplémentaires plutôt qu'un label.
